# Test Plan: Fase 1 Infrastructure Configuration

## 1. Introduction

This document outlines the test plan for verifying the core infrastructure setup corresponding to "Fase 1: Fundação e MVP" of the Modular Dashboard project. The goal is to ensure all core services are containerized correctly, can start, communicate with each other as expected, and that basic Celery task processing is functional.

## 2. Scope of Testing

-   **Container Initialization:** Verification that all services defined in `docker-compose.yml` start without errors.
    -   Services: `api` (FastAPI backend), `frontend` (React), `db` (PostgreSQL), `redis` (Redis server), `transcritor_pdf` (FastAPI service for PDF transcription - basic startup), `transcritor_pdf_worker` (Celery worker for PDF transcription - basic startup and broker connection).
-   **Service Connectivity:**
    -   Frontend to API.
    -   API to Database (PostgreSQL).
    -   API to Redis.
    -   `transcritor_pdf` service to Redis (for Celery task enqueuing or direct Redis usage).
    -   `transcritor_pdf_worker` to Redis (as Celery broker/backend).
    -   `transcritor_pdf_worker` to Database (PostgreSQL, for storing results or task-related data).
-   **Basic Celery Task Processing:**
    -   Ability of the `api` or `transcritor_pdf` service to enqueue a simple test task.
    -   Ability of the `transcritor_pdf_worker` to pick up and successfully execute this test task.
    -   Verification of task completion/result via Redis (if used as backend).
-   **Database Migrations:**
    -   Alembic migrations for the main API's database run successfully on startup.
-   **Environment Variable Loading:**
    -   Essential services load and use configurations from their respective `.env` files.

## 3. Test Strategy

Testing will primarily involve:
-   Starting the entire application stack using `docker compose up --build -d`.
-   Inspecting container logs for errors (`docker compose logs <service_name>`).
-   Using `curl` or a browser to check HTTP endpoint connectivity (e.g., API health check, frontend serving).
-   Using `docker compose exec` to run commands within containers for direct connectivity tests (e.g., `psql` to check DB connection from API and worker, `redis-cli ping` to check Redis connection from API and worker).
-   Manually triggering a test Celery task (e.g., via an API endpoint or a Python script using `send_task`).
-   Checking database schema for successful application of migrations.
